# Vibe Coding Workflow

Default mode is Lite: fast, personal, small-slice development with enough structure to avoid unclear scope and unverified work.

```text
Init -> Shape -> Plan -> Build -> Verify & Writeback -> Continue
```

## Shape

Clarify:

- Goal.
- User scenario.
- Scope.
- Non-goals.
- Completion criteria.

## Plan

Before key edits, state:

- Target outcome.
- Files likely to change.
- Risk or uncertainty.
- Verification method.

## Build

Implement one small useful slice. Avoid unrelated refactors and unrequested features.

## Verify & Writeback

Run the most relevant check. Update docs only when a decision or verification note will help future work.

## Continue

End with what changed, what was verified, risks, and the next small step.
